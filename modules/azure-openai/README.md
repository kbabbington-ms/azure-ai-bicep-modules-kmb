# Azure OpenAI Service - Dedicated Module

## Overview

This comprehensive Bicep module creates a production-ready Azure OpenAI service with advanced enterprise features including multiple model deployments, content filtering policies, customer-managed encryption, private endpoints, and comprehensive monitoring. This module is specifically designed for organizations requiring maximum security, compliance, and governance for their generative AI workloads.

## üîê Security Features

- **üîí Zero-Trust Architecture**: Private endpoints with network isolation by default
- **üõ°Ô∏è Content Safety**: Advanced content filtering with custom RAI policies
- **üîë Customer-Managed Encryption**: Azure Key Vault integration for enhanced security
- **üé≠ Identity & Access Management**: Azure AD authentication with RBAC roles
- **üìä Comprehensive Monitoring**: Audit logging, diagnostics, and security analytics
- **üåê Network Security**: IP restrictions, VNet integration, and private connectivity
- **üîê API Security**: Disabled local authentication, token-based access only

## üéØ Usage Examples

### 1. **Enterprise Production Deployment**

```bicep
module openAI 'modules/azure-openai/main.bicep' = {
  name: 'enterprise-openai'
  params: {
    openAIAccountName: 'mycompany-openai-prod'
    location: 'eastus'
    customSubDomainName: 'mycompany-openai-prod'
    
    // Model deployments
    modelDeployments: [
      {
        name: 'gpt-4o'
        model: {
          format: 'OpenAI'
          name: 'gpt-4o'
          version: '2024-08-06'
        }
        capacity: 50
        raiPolicyName: 'strict-enterprise-policy'
      }
      {
        name: 'text-embedding-3-small'
        model: {
          format: 'OpenAI'
          name: 'text-embedding-3-small'
          version: '1'
        }
        capacity: 500
        raiPolicyName: ''
      }
    ]
    
    // Maximum security
    publicNetworkAccess: 'Disabled'
    disableLocalAuth: true
    enablePrivateEndpoint: true
    vnetId: '/subscriptions/{sub}/resourceGroups/network/providers/Microsoft.Network/virtualNetworks/ai-vnet'
    privateEndpointSubnetId: '/subscriptions/{sub}/resourceGroups/network/providers/Microsoft.Network/virtualNetworks/ai-vnet/subnets/openai'
    
    // Customer-managed encryption
    enableCustomerManagedEncryption: true
    keyVaultId: '/subscriptions/{sub}/resourceGroups/security/providers/Microsoft.KeyVault/vaults/enterprise-kv'
    keyVaultKeyIdentifier: 'https://enterprise-kv.vault.azure.net/keys/openai-key/latest'
    
    // Monitoring
    enableDiagnostics: true
    logAnalyticsWorkspaceId: '/subscriptions/{sub}/resourceGroups/monitoring/providers/Microsoft.OperationalInsights/workspaces/ai-logs'
    
    tags: {
      Environment: 'Production'
      Compliance: 'SOC2,HIPAA,GDPR'
      DataClassification: 'Confidential'
    }
  }
}
```

## üîí Security Best Practices

### **1. Network Security**
- **‚úÖ Always use private endpoints** for production deployments
- **‚úÖ Disable public network access** unless specifically required
- **‚úÖ Implement IP allowlists** for any public access scenarios
- **‚úÖ Use dedicated subnets** for AI service private endpoints

### **2. Authentication & Authorization**
- **‚úÖ Disable local authentication** (API keys) in favor of Azure AD
- **‚úÖ Use managed identities** for service-to-service authentication
- **‚úÖ Implement Azure AD groups** for user access management
- **‚úÖ Apply principle of least privilege** with specific role assignments

### **3. Data Protection**
- **‚úÖ Enable customer-managed encryption** for sensitive workloads
- **‚úÖ Use HSM-backed keys** for maximum security
- **‚úÖ Implement key rotation policies** according to compliance requirements
- **‚úÖ Monitor data access patterns** through audit logs

### **4. Content Safety & Governance**
- **‚úÖ Enable advanced content filtering** with custom policies
- **‚úÖ Create industry-specific RAI policies** for compliance
- **‚úÖ Monitor content filtering events** for policy effectiveness
- **‚úÖ Regularly review and update** content filtering rules

### **5. Monitoring & Compliance**
- **‚úÖ Enable comprehensive logging** to Log Analytics
- **‚úÖ Set up security alerts** for anomalous usage patterns
- **‚úÖ Implement log retention policies** per compliance requirements
- **‚úÖ Regular security reviews** and access audits

### **Key Metrics to Monitor**

- **üîê Authentication Events**: Monitor access patterns and failures

### **Recommended Alerts**

```kql
// High error rate detection
CognitiveServicesAuditLogs
| where TimeGenerated > ago(5m)
| where ResultType startswith "4" or ResultType startswith "5"
| summarize ErrorCount = count() by bin(TimeGenerated, 1m)
| where ErrorCount > 10

// Suspicious authentication patterns
CognitiveServicesAuditLogs
| where TimeGenerated > ago(1h)
| where OperationName contains "Authentication"
| where ResultType == "Failure"
| summarize FailureCount = count() by CallerIpAddress
| where FailureCount > 20

// Content filtering violations
CognitiveServicesAuditLogs
| where TimeGenerated > ago(1h)
| where OperationName contains "ContentFilter"
| where Properties contains "blocked"
| summarize BlockedCount = count() by bin(TimeGenerated, 5m)
```
